# 01 — Domain Model

## 1. Identity

### Person
Человек как основная сущность, к которой привязана история обучения.

### UserAccount
Учетная запись человека в системе. Определяет статус доступа, роль и принадлежность к команде.

### TelegramIdentity
Привязка Telegram-аккаунта к UserAccount. Нужна для входа через бота и доставки сообщений.

### Team
Команда или группа внутри организации, к которой относится пользователь.

### Role
Роль в системе для RBAC.

---

## 2. Learning Content

### Course
Верхний уровень обучения. Объединяет сегменты. Для MVP может быть опциональным.

### Segment
Логический блок обучения (например, “Скрипт звонка” или “Работа с возражениями”).

### SegmentVersion
Версия сегмента. Нужна, чтобы изменения контента не ломали уже начатое обучение.

### Lesson
Урок внутри сегмента.

### LessonVersion
Версия урока. Нужна для безопасного обновления контента.

### MediaAsset
Файл или ссылка на медиа-контент (video, audio, document, image).

### Question
Контрольный вопрос после урока.

### Rubric
Правила оценки свободного ответа.

### QuestionKeyword
Ключевые слова / фразы для rule-based части проверки.

---

## 3. Learning Process

### Assignment
Назначение сегмента или курса пользователю.

### LessonProgress
Текущий прогресс пользователя по уроку.

### SegmentProgress
Текущий прогресс пользователя по сегменту.

### CourseProgress
Текущий прогресс пользователя по курсу.

### Attempt
Попытка ответа на вопрос урока.

### AttemptAnswer
Сырой и нормализованный текст ответа пользователя.

### AttemptScoreBreakdown
Детализация оценки: rubric score, keyword score, penalties, final score.

---

## 4. Notifications

### NotificationRule
Правило отправки уведомлений.

### ScheduledNotification
Запланированное уведомление, ожидающее отправки.

### DeliveryLog
Лог фактической отправки уведомления.

### UserNotificationPreference
Настройки уведомлений пользователя.

---

## 5. Analytics & Audit

### ActivityEvent
Сырое событие активности (открыт урок, отправлен ответ, завершен сегмент и т.д.).

### DailyUserStat
Дневной агрегат по пользователю.

### DailySegmentStat
Дневной агрегат по сегменту.

### AdminAuditLog
Журнал действий администратора.
